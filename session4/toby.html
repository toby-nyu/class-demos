CSS (Cascading Style Sheets) (this is an h3 heading)

As you can can see, this page has been styled using CSS. There is a single style rule in the HTML page <head> element, increasing the letter-spacing for the h2 heading, as well in inline styles in the main navigation icon elements - <i> - (making the graduation hats different colours).

In addition to these, there are a variety of styles rules defined in the external style sheet. Selectors are used to target the HTML. This targetting ranges from simple HTML elements like the body, h1, h2 & h3 through targeting classes and id attributes of specific elements such as - <span class="info"> which this span of text has wrapped around it</span> -, to the more complex mixing of elements and id/class and also the use of psuedo classes for the anchor elements content being additionally styled when a user hovers over, activates, or has visited a link.

Remember, id values must be unique on an HTML page, so when targetting an element using its id, there is really no need to include the element itself, unless you are intentionally adding specificity.

Remember too that there is a specific order in which you set rules for anchor pseudo selectors. and also that it is not recommended to change the font (size, family, weight, etc) for these, as this can cause the page to jump.

You can find everything you need to know about CSS from the Mozilla Developers Network, and sites like CSS tricks can be very helpful, as they often show interative examples and well have documented explanations.

Flexbox & Floats

The main menu uses Flexbox CSS - this can be viewed in the external style sheet. The <nav> element with the id "main-nav" is the container, which has the display:flexbox; rule. The children, or flexbox items are the <div> elements. The <div> element has no semantic meaning, and therefore does not inherently represent anything. Instead, it's used to group content so it can be easily styled using the class or id attributes, so perfect for this use case.

The Image element <img> for the photo above above has an id attribute - id="london-skyline". In the external CSS file the selector #london-skyline is declared. One of the rules float:right; makes the element float to the right, with text wrapping around it. This is the correct usage of the CSS float property. Using float for layout was a workaround that was used prolifically until CSS3 properties flexbox and CSS grid were developed.

Fonts (this is an h3 heading)

On this webpage, the main font for the page (except the headings - h1, h3, and span elements) is Google's Roboto, a sans-serif font. Google Fonts work by serving font files via an API, which is requested by in the head element of this HTML page. When a user visits this page, their browser downloads a CSS file from Google specifying the font, then downloads the actual font file(s) from Google's servers. These files are stored in the browser's cache, making them appear instantly on subsequent visits to pages that use the same fonts, which speeds up overall web performance. Unlike some other fonts from external providers, the @fontface rule is not required in our CSS file, as the Googe CSS file linked in the <head> does this.

The sub headings (h3) use an open-source font called Bagnard Sans Regular. You can find more about the font by clicking this text. It is licensed under the SIL Open Font License 1.1 and available for contribution, modification or download on its open-source page. It was necessary to convert the original file format, otf, to woff, for it to display in an HTML file. The original font file was downloaded from the Open Foundry website and the converted font file is stored in the same GitHub repository as the HTML and CSS files that make up this page. The CSS @font-face rule is used to reference the file in the stylesheet. This tells the browser where to find the font to render it on the website. Using downloaded font files provides greater autonomy and control over font assets, potentially improving load times and ensuring consistent display, even if external font services are unavailable. This does add an extra call to GitHub to access and download it.

The main heading (h1) uses a standard web-safe font, "Times New Roman", which is a serif font. Although it is generally considered that serif fonts reduce readability, using them for large text, such as headings, is less of an issue.

Span elements with the class info use another web-safe font, the rule is font-family:"Lucida Console", "Courier New", monospace. Span elements with the class names just use Roboto, but the font-weight value has been set to bold in the external CSS file.
